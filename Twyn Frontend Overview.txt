# Twyn Frontend Documentation

## Overview

Twyn's frontend is a modern Next.js application that provides a real-time interface for creating, running, and analyzing multi-agent simulations. It uses Server-Sent Events (SSE) for live updates and Supabase for authentication and data persistence.

---

## Technology Stack

- **Next.js 15.3** - React framework with App Router
- **React 19** - UI library
- **TypeScript 5** - Type-safe development
- **Tailwind CSS 4** - Utility-first styling with PostCSS
- **shadcn/ui** - High-quality component library built on Radix UI
- **Supabase** - Authentication & PostgreSQL database
- **Recharts** - Data visualization library
- **React Markdown** - Markdown rendering with remark-gfm
- **Vercel Analytics** - Usage tracking
- **Sonner** - Toast notifications
- **React Hook Form** - Form management with Zod validation
- **Lucide React** - Icon library
- **fast-kde** - Kernel density estimation for plots

## Hooks (`/src/hooks/`)

### useSimulationState ⭐

**The core hook for real-time simulation data.**

**Signature:**
```typescript
function useSimulationState(
  simulationId: string | null
): {
  simulationState: SimulationState | null;
  isLoading: boolean;
  error: string | null;
}
```

**How It Works:**

1. **Initial Fetch from Supabase**
   ```typescript
   // Fast initial load from database
   const state = await DatabaseService.fetchSimulationState(simulationId);
   ```

2. **Establish SSE Connection**
   ```typescript
   const eventSource = new EventSource(
     SimulationService.getStreamUrl(simulationId)
   );
   ```

3. **Listen for Events**
   ```typescript
   // Connection established
   eventSource.addEventListener('connection', (event) => {
     // Connection confirmed
   });

   // Real-time updates
   eventSource.addEventListener('update', (event) => {
     const data = JSON.parse(event.data);
     setSimulationState(data);
     
     // Sync to database
     DatabaseService.updateSimulationState(data);
   });

   // Keep-alive heartbeat
   eventSource.addEventListener('heartbeat', (event) => {
     // Connection still active
   });

   // Simulation complete
   eventSource.addEventListener('end', (event) => {
     const data = JSON.parse(event.data);
     setSimulationState(data);
     eventSource.close();
   });

   // Error occurred
   eventSource.addEventListener('error', (event) => {
     // Handle error, fallback to Supabase
     eventSource.close();
     fetchFromSupabase();
   });
   ```

4. **Automatic Cleanup**
   ```typescript
   return () => {
     eventSource.close();
   };
   ```

**Benefits:**
- Zero polling, minimal latency
- Automatic fallback to database
- Background sync for persistence
- Auto-reconnection on errors
- Memory leak prevention

## State Management Strategy

### Hybrid Approach

**1. SSE for Real-Time Updates (Primary)**
```
EventSource → Backend SSE stream
  ↓
Real-time events (update, heartbeat, end, error)
  ↓
Local state updates
  ↓
Component re-renders
```

**2. Supabase for Persistence (Backup)**
```
Initial page load → Supabase query
Background sync ← SSE updates
Fallback source ← SSE errors
Historical data ← Database queries
```

**3. React Context for Global State**
```typescript
// User authentication
<UserContext.Provider value={{ user, loading }}>
  {children}
</UserContext.Provider>

// Theme management
<ThemeProvider attribute="class" defaultTheme="system">
  {children}
</ThemeProvider>
```

**4. URL State for Navigation**
```
/sim/{uuid} → Simulation ID in URL
Shareable links
Browser history support
Deep linking
```

**Benefits:**
- Fast initial loads (Supabase)
- Real-time updates (SSE)
- Reliability (automatic fallback)
- Data persistence (database sync)
- No Redux/Zustand complexity

## Authentication Flow

### Magic Link Authentication

```
1. User visits /login
   ↓
2. Enters email in login form
   ↓
3. authService.sendMagicLink(email)
   ↓
4. Supabase sends email with link
   ↓
5. User clicks link
   ↓
6. Redirects to /auth/callback?token=...
   ↓
7. Callback route verifies token
   ↓
8. Sets session cookie (httpOnly)
   ↓
9. Redirects to /sim
   ↓
10. middleware.ts checks auth on protected routes
   ↓
11. UserContext provides user data globally
```

### Protected Routes

**Middleware (`middleware.ts`):**
```typescript
export async function middleware(request: NextRequest) {
  const { supabase, response } = await updateSession(request);
  const { data: { user } } = await supabase.auth.getUser();

  // Protect /sim routes
  if (request.nextUrl.pathname.startsWith('/sim') && !user) {
    return NextResponse.redirect('/login');
  }

  return response;
}
```

```
Verify ALLOWED_ORIGINS includes frontend URL
```

**2. Authentication Redirect Loop**
```
Solution: Clear cookies and local storage
Check Supabase URL and anon key
Verify middleware.ts logic
```

**3. Plots Not Rendering**
```
Solution: Check browser console for errors
Verify data format matches plot expectations
Ensure Recharts is properly installed
```

**4. Slow Initial Load**
```
Solution: Check Supabase query performance
Add database indexes on simulation_id
Use React Suspense for better UX
```

**5. Build Errors**
```
Solution: Delete .next and node_modules
Run npm install fresh
Check TypeScript errors: npm run lint
```
